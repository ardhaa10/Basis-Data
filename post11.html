Post 11: ERD tentang Penerapan Denormalisasi

Dipublikasikan: 1 November 2025

Apa itu ERD Denormalisasi?

Denormalisasi dalam konteks Entity Relationship Diagram (ERD) adalah proses menggabungkan tabel-tabel yang sebelumnya sudah dinormalisasi untuk mempercepat akses data. Jika normalisasi berfokus menghindari duplikasi data, maka denormalisasi justru mengizinkan pengulangan data secara terkontrol demi kinerja query yang lebih cepat.

Fungsi Denormalisasi

  • Mempercepat proses query dengan mengurangi jumlah JOIN.
  • Memudahkan pembuatan laporan dengan satu tabel besar.
  • Digunakan dalam sistem yang sering melakukan pembacaan data (read-heavy).

Kelebihan dan Kekurangan

Langkah-langkah Penerapan

  1. Buat Database Baru
    Beri nama misalnya: db_penjualan
  2. Buat Tabel Versi Normalisasi

üß± Tabel pelanggan

CREATE TABLE pelanggan (
  id_pelanggan INT AUTO_INCREMENT PRIMARY KEY,
  nama_pelanggan VARCHAR(100),
  alamat VARCHAR(200)
);

üßæ Tabel transaksi

CREATE TABLE transaksi (
  id_transaksi INT AUTO_INCREMENT PRIMARY KEY,
  id_pelanggan INT,
  tanggal DATE,
  total DECIMAL(10,2),
  FOREIGN KEY (id_pelanggan) REFERENCES pelanggan(id_pelanggan)
);
Penjelasan: Versi ini masih ter-normalisasi (3NF). Data pelanggan disimpan di tabel terpisah agar tidak ada duplikasi.

Menampilkan Masalahnya

Untuk menampilkan nama pelanggan dari tabel transaksi, kita butuh query seperti ini:

SELECT t.id_transaksi, p.nama_pelanggan, t.total
FROM transaksi t
JOIN pelanggan p ON t.id_pelanggan = p.id_pelanggan;

Query di atas menggunakan JOIN antar tabel. Jika datanya besar, maka performanya akan menurun.

Buat Versi Denormalisasi

CREATE TABLE transaksi_deno (
  id_transaksi INT AUTO_INCREMENT PRIMARY KEY,
  nama_pelanggan VARCHAR(100),
  alamat_pelanggan VARCHAR(200),
  tanggal DATE,
  total DECIMAL(10,2)
);

Pada tabel ini, data pelanggan langsung disimpan dalam tabel transaksi. Tidak perlu relasi atau foreign key.

Masukkan Data Contoh

INSERT INTO pelanggan (nama_pelanggan, alamat) VALUES
('Ardha', 'Karawang'),
('Ardha', 'Karawang');

INSERT INTO transaksi (id_pelanggan, tanggal, total) VALUES
(1, '2025-10-01', 250000),
(2, '2025-10-02', 150000);

INSERT INTO transaksi_deno (nama_pelanggan, alamat_pelanggan, tanggal, total) VALUES
('Ardha', 'Karawang', '2025-10-01', 250000),
('Sandy', 'Karawang', '2025-10-02', 150000);

Kesimpulan: Denormalisasi membantu meningkatkan kecepatan akses data dengan mengorbankan efisiensi penyimpanan. Teknik ini efektif untuk sistem laporan besar, tetapi harus digunakan secara hati-hati agar data tetap konsisten.
